- 헤더 파일은 함수,변수의 선언을 담아 여러 소스 파일에서 공유하며 목적 파일은 소스 코드가 기계어로 번역된 실제 실행 코드를 포함한다.

- 함수선언은 main 함수 앞 혹은 헤더 파일에 위치한다. 함수정의는 main 함수 아래나 별도의 소스 파일에 위치한다. 함수호출은 main 함수 내부나 다른 함수 내부에서 실제 해당 기능이 필요한 시점에 위치한다.

- exturn선언은 메모리를 할당하지 않는다. 반면 변수 선언은 변수에 대한 실제 메모리 공간을 할당한다. / cin, cout의 실제 정의는 컴파일러가 제공하는 라이브러리 파일의 내부에 존재한다.

- 컴파일러에게 사용하려는 함수나 객체의 선언 정보를 제공하여 컴파일 과정을 문제없이 진행하기 위함이다.
